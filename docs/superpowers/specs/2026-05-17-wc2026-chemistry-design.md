# WC 2026 Chemistry — Design Spec

**Date:** 2026-05-17
**Author:** Nick (with Claude)
**Status:** approved, pre-implementation
**Source paper:** Bransen & Van Haaren, *Player Chemistry: Striving for a Perfectly Balanced Soccer Team*, MIT Sloan Sports Analytics Conference 2020. [PDF](https://www.janvanhaaren.be/assets/papers/mitssac-2020-chemistry.pdf)

## 1. Goal

Produce **video-ready chart assets** for a YouTube video on WC 2026 chemistry, plus a GitHub Pages "chart browser" used to preview and download those assets. For each of the 32 FIFA WC 2026 squads, compute the **offensive chemistry** between every pair of players from their actual international matches over the past ~3 years, and visualize it as Figure-1-style pitch chemistry graphs that an editor/animator can drop into the video.

**Primary deliverable: video-ready exports**

Each nation produces, at minimum:

- **High-res pitch chemistry PNG** (4K, transparent background, no chrome).
- **Vector SVG** of the same graph, fully editable in Figma / Illustrator / After Effects.
- **Lower-third pair cards** — 1920×320 PNG per top pair (two headshots + names + JOI90 + flag), drop-in ready for video overlays.
- **Clean JSON** with every coordinate, color, and label so the partner can re-render in any tool.

Plus globally:

- **Cross-nation leaderboard** chart (PNG + SVG + JSON) of the top-50 pairs.
- **Landing grid graphic** — all 32 nations with flags and headline pair (one big asset for the video intro).

**Chart browser site (GitHub Pages)**

- Lightweight static site at `stranger9977.github.io/wc2026-chemistry/`.
- Landing page: 32-nation grid with each card linking to the nation page.
- Per-nation page: live SVG pitch viz + download buttons (PNG / SVG / JSON / pair-card pack).
- Global leaderboard page with download buttons.
- It's a browser-and-exporter, not a marketing site. Polished enough that screenshots also work directly in the video if your partner wants them, but no SEO, no analytics, no nav cruft.

**Definition of done for v1**

- Pipeline runs end-to-end on cached StatsBomb open data.
- 32 nations have squad YAMLs covering at least their likely starting XI.
- Per-nation page renders the pitch viz, the pair leaderboard, and download buttons.
- Static export script (`scripts/export.py`) produces the PNG / SVG / JSON / pair-card bundle for every nation into `exports/` ready to hand off.
- Global leaderboard renders + exports.
- Site is deployed to GitHub Pages.

**Non-goals for v1**

- JDI (defensive chemistry) — v2.
- The CatBoost prediction model for unseen pairs — every NT pair has shared international minutes by definition.
- The Team Builder MIP — managers pick squads; we visualize.
- Live in-tournament updates.

## 3. Data sources

| Source | Use | License |
|---|---|---|
| [StatsBomb Open Data](https://github.com/statsbomb/open-data) | Match events for WC 2022, Euro 2024, Euro 2020, Copa América 2024, FIFA WWC 2023, plus qualifiers and friendlies where available | Free non-commercial, attribution required |
| [statsbombpy](https://github.com/statsbomb/statsbombpy) | Python client for the above | MIT |
| [socceraction](https://github.com/ML-KULeuven/socceraction) | SPADL conversion + VAEP scoring | Apache 2.0 |
| [flag-icons](https://github.com/lipis/flag-icons) | SVG flags for all 32 nations | MIT |
| WC 2026 squads | Hand-curated YAMLs based on FA announcements, Nov 2025 / Mar 2026 call-ups, and qualifier rosters | own work |
| Headshots | Best-effort: Wikipedia Commons (CC-BY-SA) where available; initials-on-color-disc fallback otherwise | per-image |

### Data risks called out

- **Coverage is uneven.** UEFA and CONMEBOL nations are well-covered. AFC and CAF coverage is thinner — we may have decent data for Morocco, Japan, Korea, Senegal but sparse data for some first-time qualifiers. The site surfaces match count and minutes shared next to every metric so users see confidence transparently.
- **Headshot rights.** We will only use openly-licensed photos (Wikipedia Commons CC-BY-SA). Players without a freely-licensed photo get an initials-on-team-color disc fallback. Every photo carries attribution in alt-text and a credits page.
- **Final squads aren't named until ~May 26, 2026.** We bootstrap with manager's likely 26 from public trackers, swap in officials when they drop, rebuild nightly through the tournament.

## 4. Methodology adaptations (paper → internationals)

| Decision | Paper | This project | Why |
|---|---|---|---|
| Pair filter | ≥ 700 club minutes / season | **≥ 90 international minutes** across the window | NT pairs simply don't accumulate 700 mins together |
| Aggregation window | One season | **WC 2022 onward**, or last 3 years if the pair only became NT-relevant recently | Captures current-era partnerships |
| Per-90 base | Single season | Total shared minutes across the window | Same formula, longer denominator |
| Player position | Wyscout default | StatsBomb lineup position averaged across that NT's recent matches | Matches the viz target |
| Confidence | Implicit | **Explicit minutes-played + match-count badge** on every pair | Open data coverage is uneven |
| Prediction for unseen pairs | CatBoost on player features | **Out of scope for v1** | NT pairs that haven't played together aren't relevant — manager already picked them |

## 5. Pipeline data flow

```
StatsBomb JSON (statsbombpy)
        │  cached to data/raw/
        ▼
SPADL parquet (chemistry.pipeline.to_spadl)
        │  one row per on-ball action
        ▼
VAEP-scored parquet (chemistry.pipeline.score_vaep)
        │  adds vaep_value column
        ▼
Interactions parquet (chemistry.joi.enumerate_interactions)
        │  consecutive (p,q) action pairs within the same team
        ▼
Pair-match JOI parquet (chemistry.joi.joi_per_match)
        │  sums per (pair, match, ordering)
        ▼
Pair-window JOI90 (chemistry.joi.joi90_window)
        │  aggregated across window, normalised per 90 shared minutes
        ▼
Squad-filtered pair table (chemistry.squads.filter_to_squads)
        │  joined with squads/wc2026/*.yaml
        ▼
outputs/chemistry.json (chemistry.export)
        │
        ▼
site/ (static, served via GitHub Pages)
```

## 6. Modules — interfaces only

**`chemistry/ingest.py`**

- `list_open_competitions()` → list of `(competition_id, season_id, competition_name)` available in open data.
- `fetch_competition(competition_id, season_id)` → caches all matches' events JSON to `data/raw/`.
- `fetch_international_window()` → orchestrates the curated list of competitions we care about.

**`chemistry/pipeline.py`**

- `to_spadl(events_json)` → SPADL DataFrame (use `socceraction.spadl.statsbomb`).
- `score_vaep(spadl_df, model)` → SPADL DataFrame with `vaep_value` column.
- `build()` → runs the full pipeline against the cache, writes parquets to `data/spadl/` and `data/vaep/`.

**`chemistry/joi.py`**

- `enumerate_interactions(match_actions)` → DataFrame of consecutive same-team action pairs `(player_p, player_q, vaep_p, vaep_q)`.
- `joi_per_match(interactions)` → per-match per-pair JOI sum.
- `joi90_window(per_match, minutes)` → per-pair JOI90 across the window.

**`chemistry/squads.py`**

- `load_squads(dir="squads/wc2026")` → dict[nation → SquadDef].
- `filter_pairs(pairs_df, squads)` → only pairs where both players are in the same NT squad.
- `position_average(pairs_df, lineups_df)` → average pitch position per (player, nation) for viz.

**`chemistry/rank.py`**

- `per_team(pairs, squads)` → ordered list of pairs per nation.
- `global_top_n(pairs, n=50)` → cross-nation leaderboard.

**`chemistry/export.py`**

- `write_chemistry_json(per_team, global_top, path="outputs/chemistry.json")`.

## 7. Squad YAML schema

```yaml
nation: USA
nation_code: USA            # FIFA code, used for filename + flag lookup
flag_iso: us                # ISO 3166-1 alpha-2 for flag-icons
manager: Mauricio Pochettino
formation: "4-3-3"
team_color: "#bf0d3e"       # primary kit color, used for viz accent
players:
  - name: Christian Pulisic
    club: Milan
    position: RW
    headshot: pulisic.jpg   # filename in assets/headshots/, optional
  - name: Tyler Adams
    club: Bournemouth
    position: DM
```

Loader is permissive — missing headshot drops to initials disc, missing club/formation drops to defaults.

## 8. Site design (chart browser + exporter)

### 8.1 Landing page (`index.html`)

- Header: "World Cup 2026 — Chemistry" with subtitle linking to the source paper.
- 32-nation grid (4 cols × 8 rows on desktop, 2 cols on mobile). Each card:
  - Flag (SVG, top-left)
  - Nation name
  - Manager
  - Top-chemistry pair preview: two micro-headshots + names + JOI90
  - Card links to the per-nation page
- Below the grid: global leaderboard table (top 50 pairs across all nations) with nation flags inline.
- "Download landing grid graphic (PNG)" button for the video intro asset.

### 8.2 Nation page (`nation.html?nation=USA`)

- Header: large flag + nation name + manager + formation.
- **Pitch chemistry SVG** (the headline). Players positioned by their average lineup x/y, edges colored by JOI90 (green high → dark amber → red low), stroke width scales with shared minutes.
- Interactive controls:
  - "Top N" slider for showing only strongest links.
  - "Min minutes" threshold slider.
  - Hover on edge: tooltip with pair name, JOI90, shared minutes, match count.
  - Hover on player: highlight their edges, show roster card.
- **Download row** (the video-deliverable bit):
  - Pitch PNG (4K, transparent)
  - Pitch PNG (branded, team-color background)
  - Pitch SVG
  - Pair cards (.zip of lower-third PNGs)
  - Raw data JSON
- Below the pitch: ranked table of all pairs in the squad with confidence indicators.
- Squad list with headshots + clubs.
- Footer: data coverage statement, link to source paper.

### 8.3 Export formats (the part for video)

| Asset | Format | Dimensions | Background | Use |
|---|---|---|---|---|
| Pitch chemistry — clean | PNG | 3840 × 2160 | Transparent | Drop into After Effects, composite freely |
| Pitch chemistry — branded | PNG | 3840 × 2160 | Team-color gradient | Direct use as full-screen graphic |
| Pitch chemistry — vector | SVG | viewBox 1920 × 1080 | Transparent | Re-style in Figma / AE / Illustrator |
| Pair lower-third | PNG | 1920 × 320 | Transparent w/ team-color bar | Video lower-third overlay |
| Leaderboard | PNG + SVG | 1920 × 1080 | Dark theme | Section transition in video |
| Landing grid (32 nations) | PNG | 3840 × 2160 | Dark theme | Video intro / outro |
| Per-nation data | JSON | — | — | Re-render anywhere |
| Manifest | JSON | — | — | Index of all assets with paths + metadata |

All exports are deterministic — the same input data produces byte-identical output. Re-running `scripts/export.py` after a squad YAML change produces a clean diff in git.

### 8.4 Visual polish

- **Flags:** SVG flags from `flag-icons` vendored into `assets/flags/`. Rounded-corner mask.
- **Headshots:** stored in `assets/headshots/<player-slug>.jpg`, max 200px square, JPG. Fallback is an SVG initials disc rendered client-side using the nation's team color.
- **Team color accent:** kit primary color drives the page accent for that nation (link colors, focus rings, edge color base where JOI is high).
- **Dark theme** consistent with the `paper-summaries` site (same palette, same fonts).
- **Typography:** Inter for UI, JetBrains Mono for numbers/tags.

## 9. Testing

- **End-to-end:** run the pipeline against a single cached competition (WC 2022 group stage subset) and assert pair counts > 0, JOI90 distribution looks reasonable.
- **Sanity reproductions:**
  - Messi / Di María should be top-3 for Argentina.
  - Mbappé / Griezmann should be top-3 for France.
  - At least one Morocco CB pair should rank highly defensively (deferred to v2 JDI).
- **Distribution checks:** per-team JOI90 should look roughly log-normal; outliers get inspected.
- **Squad YAML validation:** schema check on every file at build time.
- **No mocks for the pipeline.** Run end-to-end against cached StatsBomb data. The cache is the test fixture.

## 10. Out of scope for v1 (explicit)

- JDI (defensive chemistry). Adds v2 once JOI is shipped and validated.
- CatBoost prediction model for unseen pairs.
- Team Builder MIP.
- Live in-tournament chemistry updates.
- Player-level (rather than pair-level) chemistry rollups.
- Manager-level comparisons.

## 11. Stretch goals if v1 is fast

- **JDI**: add defensive chemistry with the responsibility-grid approach from the paper.
- **Animation frame sequence export**: for each nation, render a 5-second build-up sequence of the pitch graph (edges fading in by JOI rank) as a PNG sequence the partner can drop straight into a timeline.
- **Form sparklines**: each player gets a 5-match form indicator on the nation page.
- **Comparison mode**: pick two nations side-by-side, diff their pitch graphs.
- **Per-match replay**: scrub through a single match's interactions live.

## 12. Open risks

1. **Open-data coverage for some CAF / AFC nations.** Mitigation: surface confidence prominently; if a nation has < 3 matches in our window we flag it on the card.
2. **Headshot licensing.** Mitigation: Wikipedia Commons + initials fallback only. Never scrape Transfermarkt/SoFIFA.
3. **Final squad churn.** Mitigation: YAMLs are dirt cheap to edit; full rebuild is < 10 minutes.
4. **VAEP model.** Use `socceraction`'s open VAEP estimator trained on what's available; document the choice in the README.

## 13. Decisions explicitly approved

- All 32 WC 2026 squads in scope.
- JOI-only for v1, JDI as v2.
- 3-year rolling window for chemistry aggregation.
- Squad mapping is hand-curated YAML.
- Headshots and flags included for visual polish; openly-licensed only.
- Primary deliverable is **video-ready asset exports** (PNG / SVG / JSON / pair-cards). GitHub Pages site exists as a browser + exporter, not as a polished public destination.
